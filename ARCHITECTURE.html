Flujo Tool-Use (Patrón Anthropic)

1 Initial Request

Usuario envía pregunta con credenciales @retarget.cl

POST /chat { "email": "luis@retarget.cl", "name": "Luis", "message": "¿Quién eres?" }

2 Tool Request

Claude analiza la pregunta y decide qué herramientas necesita

Claude: "Voy a usar get_identity para presentarme" { "type": "tool_use", "name": "get_identity", "id": "tool_1" }

3 Data Retrieval

El servidor ejecuta la herramienta localmente y obtiene los datos

runTool("get_identity", {}) → { identity: { name: "Asistente Retarget", ... } }

4 Tool Result

Se devuelve el resultado a Claude para que lo procese

{ "type": "tool_result", "tool_use_id": "tool_1", "content": "{\"identity\": {...}}" }

5 Final Response

Claude genera la respuesta final usando los datos obtenidos

Hola Luis, soy el Asistente de Retarget desarrollado por Sistemas - Retarget ❤️ Puedo coordinar al equipo, redactar mensajes, tomar screenshots y dejar respaldo de investigaciones...

∞ Loop

Si Claude necesita más datos, vuelve al paso 2. El loop continúa hasta stop_reason === "end_turn"

7 Herramientas Disponibles para Claude

get_identity

Devuelve la identidad del Asistente Retarget (nombre, firma, saludo, estilo de comunicación).

Cuándo usarla: Cuando alguien pregunta quién eres o necesitas presentarte.

get_team

Lista los miembros del equipo (Luis, Jefe 2, Periodista) con su tono, cómo escribirles y plantillas de mensaje.

Cuándo usarla: Antes de redactar un mensaje a un colaborador.

draft_message_to_member

Redacta un mensaje listo para copiar/pegar a un miembro del equipo, respetando su tono y plantilla.

Cuándo usarla: SIEMPRE que detectes un bloqueo que requiere a otra persona.

Parámetros: member_id (luis|jefe-2|periodista), vars (variables para la plantilla)

fetch_url

Descarga el contenido (HTML/texto/JSON) de una URL pública. Devuelve también el link como fuente para citar.

Cuándo usarla: Tareas investigativas que requieren datos actuales.

Devuelve: source_link, status, content_type, content (truncado a max_chars)

save_evidence

Guarda respaldo (texto/JSON) de una investigación en evidence/${project}/${view}/. Devuelve el path.

Cuándo usarla: Junto con fetch_url cuando el resultado vaya a Jefe 2 o Periodista.

Protocolo: link + screenshot + backup + reasoning_trace

validate_website

Audita un sitio web: Core Web Vitals, políticas Google Ads, SEO técnico y Mobile First. Devuelve score e issues.

Cuándo usarla: Cuando necesitas validar un sitio desplegado.

Parámetros: url, strategy (mobile|desktop)

search_lessons

Busca en la base de lecciones aprendidas (KB local) un patrón. Devuelve matches (máx 5).

Cuándo usarla: ANTES de declararte bloqueado. Busca soluciones previas.

Parámetros: query (string de búsqueda)

Equipo Retarget

Luis — Operaciones / Implementación

Tono: Directo, accionable

Cómo escribirle: Mensajes cortos. Una pregunta o pedido por mensaje. Adjuntar contexto mínimo (1-2 líneas) y el link/archivo. Cerrar con la acción concreta.

Cuándo contactarlo: Se necesita una credencial, secreto, acceso o configuración en infraestructura. Hay que ejecutar algo en su entorno. El avance está bloqueado por un permiso.

Plantilla:

Hola Luis 👋 Necesito {pedido} para avanzar con {tarea}. Contexto: {1-linea}. Link/archivo: {url} Gracias 🙌

Jefe 2 — Aprobación / Validación

Tono: Desconfiado — exige evidencia y comparativas

Cómo escribirle: Resumen ejecutivo en 3 bullets. Cada afirmación con su fuente (link). Adjuntar screenshot. Anticipar la objeción más probable y responderla en el mismo mensaje.

Cuándo contactarlo: Se necesita aprobación para un cambio que afecta producción. Hay que justificar un costo o decisión técnica. El avance depende de un OK gerencial.

Plantilla:

Jefe 2: • Qué propongo: {propuesta} • Por qué (con fuente): {razon} — {link_fuente} • Riesgo si NO se hace: {riesgo} • Evidencia adjunta: {screenshot_path} Alternativa considerada y descartada: {alternativa} ({por_que_no}).

Periodista — Comunicación Externa / Contenido

Tono: Exige certeza, datos verificables, fuentes citadas

Cómo escribirle: Cada dato con su fuente entre paréntesis. Si no hay fuente, decir explícitamente "sin verificar". Distinguir hecho vs opinión. Entregar versión corta (140 chars) y versión larga.

Cuándo contactarlo: Hay que comunicar algo al público. Se redacta un texto institucional. Se necesita un dato verificado para una nota.

Plantilla:

Periodista: TEMA: {tema} DATO PRINCIPAL: {dato} (fuente: {url}) DATOS DE APOYO: - {dato2} (fuente: {url2}) - {dato3} (fuente: {url3}) VERSIÓN CORTA (≤140): {tweet} VERSIÓN LARGA: {parrafo}

Deployment en Google Cloud Run

1 Repositorio GitHub

Repo: https://github.com/SistemasRetarget/MCP

Rama: main

Archivos clave:

  • Dockerfile — Imagen Node.js + Rust
  • cloudbuild.yaml — Pasos de build y deploy
  • http-wrapper.mjs — Servidor HTTP
  • chat.mjs — Tool-use loop
  • mcp-subagents-config.json — Configuración
  • contracts/team.json — Equipo

2 Cloud Build

Proyecto: retarget-mcp (ID: 201530409487)

Pasos:

  1. Build Docker image: europe-west1-docker.pkg.dev/retarget-mcp/.../mcp-server:latest
  2. Push a Artifact Registry
  3. Deploy a Cloud Run con secrets inyectados

3 Cloud Run Service

Nombre: cmsretargetv1

URL: https://cmsretargetv1-201530409487.europe-west1.run.app

Configuración:

  • Región: europe-west1
  • Memoria: 512 MB
  • CPU: 1
  • Min instances: 0 (cold start)
  • Max instances: 2
  • Timeout: 3600s
  • Unauthenticated: Sí (auth por email @retarget.cl en la app)

4 Secrets Manager

Secretos inyectados en runtime:

  • ANTHROPIC_API_KEY — Para llamadas a Claude (obligatorio)
  • PAGESPEED_API_KEY — Para validar Core Web Vitals (opcional)

Cómo crear el secret:

echo -n "tu-api-key" | gcloud secrets create ANTHROPIC_API_KEY --data-file=- gcloud secrets add-iam-policy-binding ANTHROPIC_API_KEY \ --member="serviceAccount:201530409487-compute@developer.gserviceaccount.com" \ --role="roles/secretmanager.secretAccessor"

5 Redeploy

Comando:

gcloud builds submit --config=cloudbuild.yaml /Users/spam11/github/MCP

El build trigger se activa automáticamente con cada push a main.
