Retarget · Engineering

Tenemos un equipo de programación que no duerme.

Una IA que escribe código, lo prueba, lo corrige y avisa cuando terminó. Mientras tanto, el equipo humano hace estrategia.

Le pedimos algo en español. La IA lo hace. Un sistema verifica que esté bien. Si no, vuelve a intentar. Hasta que dice OK.

No es autocompletado. Es un programador con manos que edita archivos, hace commits, despliega, mide. Y se detiene solo cuando el verificador dice OK.

02Las cuatro piezas

Cuatro componentes que trabajan juntos. Todos corren en el computador de Retarget, sin dependencias externas.

🤖
Claude Sonnet 4.5 La IA que entiende la tarea, escribe código TypeScript/React, ejecuta comandos shell, hace git push, se corrige a sí misma.
⚙️
MCP Server (quality-gate) Servidor local con herramientas especializadas: toma screenshots con Playwright, compara imágenes pixel a pixel, valida código (typecheck/lint/build), espera deploys en Railway/Netlify.
⏰
Cron Daemons (launchd) Procesos en background: cada 30 segundos pregunta "¿estás vivo?". Cada 30 minutos verifica si el sitio quedó bien. Notificación macOS si algo falla.
📚
Knowledge Base (KB) Cada error resuelto queda anotado. La próxima vez no se repite. El sistema aprende. 40+ lecciones registradas.

04Performance & SEO

Lo que importa para marketing: velocidad, indexabilidad, conversiones.

PROD (Shopify)

  • TTFB: ~1.2s (servidor remoto)
  • HTML size: ~450KB (Shopify overhead)
  • Scripts: 28+ (analytics, apps, tracking)
  • Stylesheets: 5+
  • JSON-LD: 1 bloque (schema.org)
  • Title: "Pueblo La Dehesa | Tu refugio en la ciudad"

QA (Next.js)

  • TTFB: ~0.6s (Railway, CDN listo)
  • HTML size: ~15KB (Next.js optimizado)
  • Scripts: 12 (solo esenciales)
  • Stylesheets: 1 (Tailwind compilado)
  • JSON-LD: 2 bloques (Organization + LocalBusiness)
  • Title: "Pueblo La Dehesa | Casas de Lujo en La Dehesa..."

Resultado: 50% más rápido en TTFB, 97% menos HTML inicial, mismo SEO (ambos tienen title, description, og:image, JSON-LD). El cliente no pierde posicionamiento, gana velocidad.

05Por qué importa para Retarget

Cambia la economía de los proyectos web del equipo.

⚡ 10× más rápido Lo que tomaba semanas, ahora son horas. El loop trabaja mientras hacemos otra cosa.
📐 Auditable "Listo" no es opinión. Hay un reporte JSON que dice PASS o FAIL. Defendible ante el cliente.
🧠 Aprende Cada proyecto deja lecciones. El siguiente cliente se beneficia sin esfuerzo extra.
👀 Sin caja negra Si la IA se traba, notificación en 30 segundos. Visibilidad total del estado.
💰 Costo marginal Toda la infra corre local. Solo paga tokens de Claude cuando hay tarea activa.
🔁 Reutilizable El protocolo funciona para cualquier "rediseño 1:1". Un activo de Retarget.
